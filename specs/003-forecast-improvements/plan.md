# Implementation Plan: Forecast Accuracy Improvements

**Branch**: `003-forecast-improvements` | **Date**: 2026-03-04 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/003-forecast-improvements/spec.md`

## Summary

Improve forecast accuracy through five capabilities: (1) per-hour SLR-based snowfall calculation replacing raw API snowfall, (2) forecast confidence calibration based on lead time, (3) multi-model weather queries with consensus detection, (4) NWS Area Forecast Discussion integration for US resorts, and (5) high-resolution model selection for near-range forecasts. Changes span the weather fetching layer, domain types, evaluation prompts, and the detection/pipeline integration.

## Technical Context

**Language/Version**: Go 1.23+
**Primary Dependencies**: `google.golang.org/genai` (Gemini), `modernc.org/sqlite`, `net/http` (weather APIs)
**Storage**: SQLite via `modernc.org/sqlite` — single file, pure Go
**Testing**: `go test ./...` with sociable tests at boundaries + edge-case unit tests
**Target Platform**: Linux (Docker on Unraid), also macOS for development
**Project Type**: CLI tool (cron-triggered pipeline)
**Performance Goals**: Complete full pipeline run (scan + evaluate + post) within 5 minutes for all regions
**Constraints**: No CGO. Open-Meteo is free/unauthenticated. NWS requires User-Agent header. Gemini API key required.
**Scale/Scope**: ~10-15 regions, ~30 resorts, 2x daily runs
